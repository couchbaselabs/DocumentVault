Demonstration Overview

This interactive walkthrough guides you through illustrating the real-time cloud-to-edge synchronization, offline-first operation, and embedded vector search capabilities of Couchbase Lite (CBL) inside the iOS DocumentVault app alongside our Capella App Services instance.

🛡️ Enforcing RAG Governance Boundaries

Standard vector RAG matches search queries globally based on semantic similarity. In secure data environments (like banking or legal casework), this introduces data leakages and cross-matter pollution. To mitigate this, the app utilizes composite scalar-vector query pre-filtering to restrict search ranges prior to executing semantic scans.

Approach Execution Order Compliance Risk
Unscoped (Standard RAG) ANN Vector scan → Post-filter results High (Retrieves outdated periods or wrong clients)
Scoped (Composite Pre-Filtering) Enforce Scope (Folder/Client ID) → ANN scan Zero (Strict boundary isolation verified at indexing level)

📖 The Narrative: Core Problems We Are Solving

💰 1. Zero Cloud Compute & Token Costs

Traditional mobile vector retrieval requires constant cloud vector databases (e.g., Pinecone/pgvector) and high token network request costs. Couchbase Lite performs all float vector comparison math 100% on-device within the local SQLite engine, eliminating cloud query bills.

🔒 2. Data Sovereignty & Absolute Privacy

Law firms, courtrooms, and banks cannot leak sensitive documents to third-party cloud vector search caches. By keeping embeddings on-device and using Apple Intelligence locally, secure casework documents never leave the physical device boundary.

📶 3. Air-Gapped Offline Resilience

If a lawyer is in a basement vault or a courtroom with no cellular signal, standard RAG apps break completely. Couchbase Lite ensures the entire RAG pipeline—embeddings, indices, document lookups, and summaries—is fully functional while offline.

🔄 4. Real-time Reactive Freshness

Vector stores usually rely on scheduled batch jobs, meaning search results are stale. With CBL database change listeners, the moment a new annotation or review note syncs in from another device, the local vector index is automatically updated in the background.

Trigger simulated Jane Email (Matter AN-01) Action
Demonstrate how incoming email updates dynamically alter the vector search indexes and AI summaries of related case documents.
  • In the iPad app, go to the Profile tab.
  • Tap "Simulate Email from Jane (Matter AN-01)".
  • This automatically creates a new .eml document, logs an emailCaptured custody chain audit event on the PwC case matter document, and saves a linked annotation update.
⚡ Couchbase Reactive Sync: The database change listeners immediately catch the new annotation, schedule background re-summarization, and generate a new unit-normalized embedding of the tax document including Jane's deferred tax updates.
5
Verify Dynamic AI Updates & Vector Search Verification
Verify the changes and test boundary leakage constraints inside the Agent Chat:
  • Open the **Documents List**: You will find Jane's email saved as a previewable document with an envelope icon.
  • Open the **PwC Tax Return file**:
    • The **Summary** section now lists PwC's deferred tax negligence.
    • The **Custody** tab shows the automated email capture audit trail.
  • Open the **Agent Chat** tab:
    • Showcase Baseline (No RAG): Toggle the "Couchbase RAG" switch to Disabled. Ask: "Was there an important matter about PwC negligence?". The LLM will decline or fail to answer, proving that it has no knowledge of local files or private emails.
    • Showcase Couchbase RAG: Toggle the "Couchbase RAG" switch to Enabled. Re-run the query. The LLM will provide a detailed, accurate response citing Jane's email and the tax filings.
    • Showcase Context Scoping: Set the "Context Scope" picker to a folder outside of the target matter (e.g. choose "Legal Contracts" instead of "Financial Records & Audits"). Re-run the query. The LLM will report that it cannot find any information, demonstrating secure data partition and context boundaries (critical for compliance and client data segregation).
