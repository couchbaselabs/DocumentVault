Topology & Folder Specifications

Collections:
  • documents: Metadata, vectors, FTS summaries
  • folders: Dir hierarchies
  • annotations: Highlights, review comments, and vector steering facts
  • profile: User profiles & role settings
  • senders: Ingestion pipeline identities
  • threads: Message threads
📱 In-App Couchbase Lite Structure
Local Database: DocumentVaultDB.cblite2
Key Indexes:
  • vector_idx: 512-dim embedding index with Cosine metric similarity
  • fts_idx: Full-text indexing for name, textContent, summary, and tags
  • folder_idx, updated_idx: Standard B-Tree value search indexes
All file attachments are stored locally as offline CBL Blobs.

Search & Relevance Feedback Flows
🔤 Keyword Search
  • 1 User types in SearchView (.keyword mode)
  • 2 FTS MATCH on fts_idx (textContent · summary · tags)
  • 3 Fallback: LIKE on name if FTS returns 0
  • 4 Results grouped: full / metadata-only / deleted
🧠 Hybrid Search (RRF)
  • 1 Rocchio adjusts query embedding from session feedback
  • 2 FTS ranked list (keyword) + Vector ranked list (semantic) run in parallel
  • 3 RRF merge: score += 1 / (60 + rank) for each list
  • 4 Persistent searchBoost added per document
  • 5 Top-N sorted by combined score
👍 Relevance Feedback
  • 1 User swipes → markRelevant(doc, query)
  • 2 Session: Rocchio centroid updated in memory
  • 3 Persistent: searchBoost += 0.15 (max 1.0)
  • 4 feedbackTags: query tokens appended to doc.tags
  • 5 Re-embed triggered → vector shifts toward query
  • 6 Search re-runs immediately
📥 Ingest Pipeline
  • 1 File URL or camera → ContentStorageService (copy + SHA-256)
  • 2 Duplicate / restore check via contentHash
  • 3 VaultDocument created, CBL blob attached
  • 4 TextExtractionService → textContent
  • 5 ImageAnalysisService (images) → tags + color
  • 6 DocumentSummaryService → summary + topics + category
  • 7 EmbeddingManager → generates L2-normalized 512-dim vector from cleaned filename + AI summary + active annotations (or text/tags fallback) → CBL
